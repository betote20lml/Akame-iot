Trae Modifier, que es clave en Compose.

------------------

@Composable

Esto es una anotación (como una etiqueta) que le dice a Compose:

“Esta función no es una función normal. Esta función produce UI.”

Sin @Composable, no puedes poner dentro cosas como Text(), Button(), Box(), etc.

Analogía JS:
Es como decir “esto es un componente React”.
------------------

fun LoginScreen(onLoginSuccess: () -> Unit) {
fun

Significa “función”.
(En JS sería function.)

LoginScreen

Nombre de la función.
Por convención, Composables van con PascalCase (mayúscula al inicio), como componentes.

(onLoginSuccess: () -> Unit)

Esto es un parámetro que le pasas a tu pantalla.

onLoginSuccess es el nombre.

: () -> Unit es el tipo.

Qué significa () -> Unit:

() = no recibe parámetros

-> = “regresa”

Unit = “nada” (como void en JS/TS)

O sea:

onLoginSuccess es una función que no recibe nada y no regresa nada.

modifier = Modifier.fillMaxSize()

Analogía CSS:

contentAlignment = Alignment.Center
